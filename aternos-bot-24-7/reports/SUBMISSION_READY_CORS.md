## Title
Stripe API CORS Misconfiguration - Arbitrary Origin Reflected with Credentials

## Severity
HIGH

## Summary
The Stripe API at https://api.stripe.com/v1/customers allows arbitrary cross-origin requests from any origin when credentials are included. The server reflects the Origin header value back in the Access-Control-Allow-Origin response header, and Access-Control-Allow-Credentials is set to true.

## Steps to Reproduce

1. Send a preflight OPTIONS request to https://api.stripe.com/v1/customers:
```bash
curl -I -X OPTIONS 'https://api.stripe.com/v1/customers' -H 'Origin: https://evil-test.com' -H 'Access-Control-Request-Method: GET'
```

2. Observe the response headers:
```
HTTP/2 204
access-control-allow-credentials: true
access-control-allow-methods: GET, HEAD, PUT, PATCH, POST, DELETE
access-control-allow-origin: https://evil-test.com
access-control-max-age: 300
vary: Origin
```

## Impact
An attacker can craft a malicious webpage that makes XMLHttpRequest/fetch calls to the Stripe API on behalf of logged-in users, potentially stealing:
- Billing information
- Customer data
- Payment method details

## Remediation
Implement strict CORS policy by:
1. Using a whitelist of allowed origins
2. Not reflecting arbitrary Origin headers
3. Only allowing specific trusted domains in Access-Control-Allow-Origin
