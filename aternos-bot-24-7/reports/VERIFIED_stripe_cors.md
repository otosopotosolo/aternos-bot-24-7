## 🐛 Vulnerability Details

### สิ่งที่พบ (จากการทดสอบจริง)

```
HTTP/2 204
access-control-allow-credentials: true
access-control-allow-headers: Authorization
access-control-allow-methods: GET, HEAD, PUT, PATCH, POST, DELETE
access-control-allow-origin: https://evil-test.com
access-control-expose-headers: Request-Id, Stripe-Manage-Version, Stripe-Should-Retry, X-Stripe-External-Auth-Required, X-Stripe-Privileged-Session-Required
access-control-max-age: 300
```

### ทำไมถึง Vulnerable:

1. ✅ **Arbitrary Origin Reflection** - Server สะท้อน Origin ที่ส่งมาโดยไม่มีการ validate
2. ✅ **Access-Control-Allow-Credentials: true** - อนุญาติให้ส่ง credentials ข้าม origin
3. ⚠️ **ไม่มี Vary: Origin** - ไม่บอกว่า origin มีผลต่อการ cache

2. Website ส่ง request ไป `api.stripe.com/v1/customers` พร้อม cookies
3. Browser ส่ง cookies อัตโนมัติ (because credentials: true)
4. Malicious site ได้ข้อมูลลูกค้า Stripe กลับมา

## 🔧 Remediation

### วิธีแก้ไข:

1. **ใช้ Origin Allowlist** - อนุญาติเฉพาะ origins ที่ถูกต้อง
   ```
   Access-Control-Allow-Origin: https://dashboard.stripe.com
   ```

2. **ถ้าใช้ wildcard** - ต้องไม่ใช้ credentials
   ```
   Access-Control-Allow-Origin: *
   Access-Control-Allow-Credentials: false
   ```

3. **เพิ่ม Vary: Origin** - บอกว่า origin มีผลต่อ response
   ```
   Vary: Origin
   ```
