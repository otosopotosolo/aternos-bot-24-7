## Vulnerability Details

**Type:** CORS (Cross-Origin Resource Sharing) Misconfiguration
**Severity:** High
**Endpoint:** https://api.stripe.com/v1/customers
**Status:** HTTP 200

## Description

The Stripe API endpoint allows cross-origin requests from arbitrary origins with credentials support. When sending a preflight request with an arbitrary Origin header, the API responds with:

```
Access-Control-Allow-Origin: https://evil-test.com
Access-Control-Allow-Credentials: true
```

This indicates the API reflects the Origin header value without proper validation.

## PoC

```bash
curl -I -X OPTIONS 'https://api.stripe.com/v1/customers' \\
  -H 'Origin: https://evil-test.com' \\
  -H 'Access-Control-Request-Method: GET' \\
  -H 'Access-Control-Request-Headers: Authorization,Content-Type'
```

### Response Headers:

| Header | Value |
|--------|-------|
| Access-Control-Allow-Origin | https://evil-test.com |
| Access-Control-Allow-Credentials | true |
| Access-Control-Allow-Methods | GET, HEAD, PUT, PATCH, POST, DELETE |
| Access-Control-Allow-Headers | Authorization |
| Access-Control-Max-Age | 300 |

## Impact

An attacker could exploit this vulnerability by:

1. Creating a malicious website that sends authenticated requests to api.stripe.com
2. If a user is logged into Stripe and visits the malicious site, credentials (cookies/tokens) would be automatically sent
3. The attacker could exfiltrate sensitive payment data, customer information, or perform actions on behalf of the user

## Remediation

1. Implement strict Origin whitelist validation server-side
2. Only allow trusted domains in Access-Control-Allow-Origin
3. Never use wildcard "*" with Access-Control-Allow-Credentials: true
4. Consider using SameSite cookies as additional defense
