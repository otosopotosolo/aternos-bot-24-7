## Title
Discord API CORS Misconfiguration - Arbitrary Origin Reflected with Credentials

## Severity
HIGH

## Summary
The Discord API at https://discord.com/api/v10/users/@me allows arbitrary cross-origin requests from any origin with credentials enabled. The server reflects the Origin header value back in the Access-Control-Allow-Origin response header.

## Steps to Reproduce

1. Send a preflight OPTIONS request to https://discord.com/api/v10/users/@me:
```bash
curl -I -X OPTIONS 'https://discord.com/api/v10/users/@me' -H 'Origin: https://evil-test.com' -H 'Access-Control-Request-Method: GET'
```

2. Observe the response headers:
```
HTTP/2 200
access-control-allow-origin: https://evil-test.com
access-control-allow-credentials: true
access-control-allow-methods: POST, GET, PUT, PATCH, DELETE
```

## Impact
An attacker can craft a malicious webpage that makes XMLHttpRequest/fetch calls to the Discord API on behalf of logged-in users, potentially stealing:
- User profile information
- Direct messages
- Server memberships
- Payment and subscription data

## Remediation
Implement strict CORS policy by:
1. Using a whitelist of allowed origins (*.discord.com, *.discord.gg)
2. Not reflecting arbitrary Origin headers
3. Implementing proper origin validation
