## Summary

A CORS misconfiguration was discovered in the aws_s3 API. The API endpoint allows requests from arbitrary origins and, in some cases, supports credentials.

---

## Steps to Reproduce

1. Send a preflight OPTIONS request:
```bash
curl -I -X OPTIONS 'https://s3.amazonaws.com/profile' \
  -H 'Origin: https://evil-test-cors.com' \
  -H 'Access-Control-Request-Method: GET' \
  -H 'Access-Control-Request-Headers: Authorization,Content-Type'
```

2. Observe the response headers:
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Credentials: 
Access-Control-Allow-Methods: HEAD, DELETE, POST, GET, OPTIONS, PUT, PATCH
```

## Impact

MEDIUM severity - Wildcard CORS policy without credentials. Allows:
- Cross-origin access to public data
- Potential information disclosure
- CSRF protection bypass

---

## Remediation

1. Use a strict whitelist of allowed origins
2. Don't use `*` wildcard for `Access-Control-Allow-Origin` with `Access-Control-Allow-Credentials: true`
3. Validate Origin header server-side
4. Use CORS properly configured for your specific domain(s)
