## Title
Twilio API CORS Misconfiguration - Arbitrary Origin Reflected with Credentials

## Severity
HIGH

## Summary
The Twilio API at https://api.twilio.com/2010-04-01/Accounts.json allows arbitrary cross-origin requests from any origin with credentials enabled. The server reflects the Origin header value back in the Access-Control-Allow-Origin response header.

## Steps to Reproduce

1. Send a preflight OPTIONS request to https://api.twilio.com/2010-04-01/Accounts.json:
```bash
curl -I -X OPTIONS 'https://api.twilio.com/2010-04-01/Accounts.json' -H 'Origin: https://evil-test.com' -H 'Access-Control-Request-Method: GET'
```

2. Observe the response headers:
```
HTTP/1.1 200 OK
Access-Control-Allow-Credentials: true
Access-Control-Allow-Origin: https://evil-test.com
Access-Control-Allow-Methods: POST,GET,DELETE,OPTIONS
Vary: Origin
```

## Impact
An attacker can craft a malicious webpage that makes XMLHttpRequest/fetch calls to the Twilio API on behalf of logged-in users, potentially stealing:
- Account information
- Phone numbers
- Call logs
- SMS history

## Remediation
Implement strict CORS policy by:
1. Using a whitelist of allowed origins (*.twilio.com)
2. Not reflecting arbitrary Origin headers
3. Validating origins against a predefined whitelist
