## 🐛 Vulnerability Details

### สิ่งที่พบ (จากการทดสอบจริง):

```
HTTP/2 200
access-control-allow-origin: https://evil-test.com
access-control-allow-credentials: true
access-control-allow-methods: POST, GET, PUT, PATCH, DELETE
access-control-allow-headers: Content-Type, Authorization, X-Audit-Log-Reason, ...
```

### ทำไมถึง Vulnerable:

1. ✅ **Arbitrary Origin Reflection** - Server สะท้อน Origin ที่ส่งมาโดยไม่ตรวจสอบ
2. ✅ **Access-Control-Allow-Credentials: true** - อนุญาติให้ส่ง cookies ข้าม origin
3. ✅ **หลาย methods** - รองรับ POST, GET, PUT, PATCH, DELETE

## 📋 Proof of Concept

## 💥 Impact

### ความรุนแรง: HIGH (สูงกว่าเพราะ Discord มีข้อมูล sensitive)

| Factor | Score | Description |
|--------|-------|-------------|
| Attack Vector | Network | สามารถโจมตีจาก internet ได้ |
| Privileges Required | None | ไม่ต้องมีสิทธิ์ก่อน |
| User Interaction | Required | ต้องมี user ที่ logged in เข้าชม malicious page |
| Confidentiality | High | ข้อมูล Discord user, guilds, messages, friends |

### Attack Scenario:

1. User ที่ login Discord เข้าชม malicious website
2. Website ส่ง request ไป `discord.com/api/v10/users/@me` พร้อม cookies
3. Malicious site ได้ข้อมูล profile, guilds, และ permissions กลับมา

---

## 🔧 Remediation

1. **ใช้ Origin Allowlist** - อนุญาติเฉพาะ origins ของ Discord
2. **ตรวจสอบ Origin** - ถ้าไม่อยู่ใน allowlist ให้ return error หรือไม่ส่ง CORS headers
