## Vulnerability Details

**Type:** CORS (Cross-Origin Resource Sharing) Misconfiguration
**Severity:** High
**Endpoint:** https://api.sendgrid.com/v3/templates
**Status:** HTTP 200

## Description

The SendGrid API endpoint allows cross-origin requests from arbitrary origins with credentials support. When sending a preflight request with an arbitrary Origin header, the API responds with:

```
Access-Control-Allow-Origin: https://evil-test.com
Access-Control-Allow-Credentials: true
```

This indicates the API reflects the Origin header value without proper validation.

## PoC

```bash
curl -I -X OPTIONS 'https://api.sendgrid.com/v3/templates' \\
  -H 'Origin: https://evil-test.com' \\
  -H 'Access-Control-Request-Method: GET' \\
  -H 'Access-Control-Request-Headers: Authorization,Content-Type'
```

### Response Headers:

| Header | Value |
|--------|-------|
| Access-Control-Allow-Origin | https://evil-test.com |
| Access-Control-Allow-Credentials | true |
| Access-Control-Allow-Methods | HEAD, GET, PUT, POST, DELETE, OPTIONS, PATCH |
| Access-Control-Allow-Headers | AUTHORIZATION, Content-Type, On-behalf-of, x-sg-elas-acl |
| Access-Control-Max-Age | 21600 |

## Impact

An attacker could exploit this vulnerability by:

1. Creating a malicious website that sends authenticated requests to api.sendgrid.com
2. If a user is logged into SendGrid and visits the malicious site, credentials (API keys/tokens) would be automatically sent
3. The attacker could access email marketing data, contact lists, templates, and send emails on behalf of the user

## Remediation

1. Implement strict Origin whitelist validation server-side
2. Only allow trusted domains in Access-Control-Allow-Origin
3. Never use wildcard "*" with Access-Control-Allow-Credentials: true
4. Validate Origin header against a list of allowed origins
